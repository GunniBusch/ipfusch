# ipfusch

`ipfusch` is a modern Rust network benchmark tool with:
- native protocol and fixed-size packet header (`session_id`, `stream_id`, `seq`, `send_mono_ns`)
- TCP / UDP / QUIC transports
- multi-stream benchmarking
- one binary for CLI and TUI (`--tui`)
- JSON and JSONL output modes
- sweep mode with ranked recommendations
- bash/zsh completion generation

## Quick Start (No Install, curl only)

Run `ipfusch` directly from the latest GitHub Release without installing anything locally:

```bash
curl -fsSL https://raw.githubusercontent.com/GunniBusch/ipfusch/main/scripts/run-ipfusch.sh | bash -s -- --help
```

Run a benchmark via curl launcher:

```bash
curl -fsSL https://raw.githubusercontent.com/GunniBusch/ipfusch/main/scripts/run-ipfusch.sh \
  | bash -s -- run --host 127.0.0.1:5201 --transport tcp --streams 4 --duration 10s
```

Optional environment variables for the launcher:
- `IPFUSCH_REPO` (default: `GunniBusch/ipfusch`)
- `IPFUSCH_VERSION` (default: latest release tag)

## Local Build

```bash
cargo build --release
```

## Server

```bash
cargo run -p ipfusch-cli -- server --bind 0.0.0.0:5201 --transport tcp
cargo run -p ipfusch-cli -- server --bind 0.0.0.0:5201 --transport udp
cargo run -p ipfusch-cli -- server --bind 0.0.0.0:5201 --transport quic
```

`--transport all` starts:
- TCP on `port`
- UDP on `port+1`
- QUIC on `port+2`

## Run

```bash
cargo run -p ipfusch-cli -- run --host 127.0.0.1:5201 --transport tcp --streams 4 --duration 10s
cargo run -p ipfusch-cli -- run --host 127.0.0.1:5201 --transport udp --streams 8 --rate 500m --json
cargo run -p ipfusch-cli -- run --host 127.0.0.1:5201 --transport quic --tui
```

## Sweep

```bash
cargo run -p ipfusch-cli -- sweep \
  --host 127.0.0.1:5201 \
  --transport tcp \
  --streams 1,2,4,8 \
  --rates auto,100m,500m,1g \
  --objective balanced
```

## Completion

```bash
cargo run -p ipfusch-cli -- completion bash > ipfusch.bash
cargo run -p ipfusch-cli -- completion zsh > _ipfusch
```

## Test

```bash
cargo test
```

Integration tests include consistent transport checks for TCP/UDP/QUIC and token-auth validation.

## CI/CD and Releases

### CI (`.github/workflows/ci.yml`)

Runs on pull requests and pushes to `main`:
- `cargo fmt --all -- --check`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`

### Release (`.github/workflows/release.yml`)

Supports two release paths:
- push a semver tag (`v*`) to trigger a release pipeline
- manually dispatch the workflow with a tag (for example `v0.1.0`)

Release pipeline behavior:
- validates tag format
- builds binaries for:
  - `x86_64-unknown-linux-gnu`
  - `aarch64-apple-darwin`
- creates release artifacts:
  - `ipfusch-<tag>-<target>.tar.gz`
  - `ipfusch-<tag>-<target>.tar.gz.sha256`
- publishes assets to GitHub Releases with autogenerated notes
